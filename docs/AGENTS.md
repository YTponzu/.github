# エージェント共有ガイド

AIエージェントにプロジェクトの文脈・ルールを共有するための設計ガイド。

---

## 基本方針

- 各プロジェクトのルートに **`AGENTS.md`** を置く
- エージェントは**開発セッション開始時に必ず読む**
- 「何をするプロジェクトか」「何に気をつけるか」を簡潔にまとめる
- 長くなりすぎない（詳細は `docs/` を参照させる）

---

## AGENTS.md の構成テンプレート

```markdown
# AGENTS.md

## このプロジェクトについて
（1〜3行でプロジェクトの目的・概要）

## 技術スタック
（使用技術の一覧）

## セッション開始時にやること
1. このファイルを読む
2. docs/ のドキュメントを確認する
3. 直近のブランチ・PRの状態を確認する（git status / git log）

## ルール（必読）
- CONTRIBUTING.md — 開発の基本フロー
- docs/ARCHITECTURE.md — ディレクトリ構成・設計方針
- docs/DESIGN.md — デザインルール
- docs/TESTING.md — テスト方針・TDDサイクル

## やってはいけないこと
（プロジェクト固有のNG事項）

## よくあるタスクのやり方
（定型作業のコマンド・手順）
```

---

## 効果的な AGENTS.md を書くコツ

### ✅ 良い書き方

- **短く・具体的に** — 長文より箇条書き
- **NG事項を明示する** — 「〜しない」を書くと事故が減る
- **コマンドを書く** — 「開発サーバーは `npm run dev`」のように即使えるものを
- **定期的に更新する** — プロジェクトの状態が変わったら更新する

### ❌ 避けること

- 全ての詳細をここに書く（詳細は `docs/` に任せる）
- 古い情報を放置する
- 抽象的な説明だけ（「きれいなコードを書く」など）

---

## エージェントへの指示パターン

### セッション開始時

```
AGENTS.md と docs/ のドキュメントを読んでから作業を始めてください。
```

### タスク依頼時

```
[タスクの説明]
AGENTS.md のルールに従って実装し、PRを作成してください。
```

### レビュー依頼時

```
DESIGN.md のルールに従っているか確認してください。
特に CSS 変数の使い方とコンポーネントの分割が適切かを見てください。
```
