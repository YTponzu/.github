# GitHub ルール（共通）

このドキュメントは YTponzu の全リポジトリに適用される共通ルールです。

## ブランチ戦略（GitHub Flow）

| ブランチ | 用途 |
|----------|------|
| `main` | 常にデプロイ可能な状態を保つ |
| `feature/xxx` | 機能追加 |
| `fix/xxx` | バグ修正 |
| `docs/xxx` | ドキュメントのみの変更 |

- `main` への直接プッシュは禁止
- 作業は必ずブランチを切ってから開始する

## コミット規約（Conventional Commits）

```
<type>: <概要>
```

### type 一覧

| type | 使うタイミング |
|------|--------------|
| `feat` | 新機能の追加 |
| `fix` | バグ修正 |
| `docs` | ドキュメントのみの変更 |
| `chore` | 依存パッケージ更新・設定変更など |
| `refactor` | 動作を変えないコードの整理 |
| `style` | フォーマット・見た目の微調整 |
| `test` | テストの追加・修正 |

### 例

```
feat: ログイン機能を追加
fix: ダッシュボードの表示崩れを修正
docs: ARCHITECTURE.md を追加
chore: 依存パッケージを最新バージョンに更新
```

## PR ルール

- タイトルはコミット規約と同じ形式（例：`feat: xxx を追加`）
- 本文に変更内容・確認事項を記載
- セルフマージOK（小規模チームのため）
- マージ後はブランチを削除する
